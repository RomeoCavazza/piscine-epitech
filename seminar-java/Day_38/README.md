## Objectifs
- Découvrir les Design Patterns fondamentaux.  
- Comprendre les principes SOLID de conception.  
- Appliquer les patterns Factory, Composite, Observer, Decorator.  

## Actions
- **Ex01-Ex02 - Factory Pattern** : Usine de jouets (Toy, TeddyBear, Gameboy, Factory, Elf).  
- **Ex03 - Composite Pattern** : Structure arborescente pour les phrases (Sentence, Word, SentenceComposite).  
- **Ex04 - Observer Pattern** : Notification en temps réel des commandes (Observable, Observer, Order, Customer).  
- **Ex05 - Decorator Pattern** : Enrichissement dynamique des guerriers (Warrior, BasicWarrior, KingWarrior, Shield, FireSword).  

## Leçons
- **Factory Pattern** : Encapsulation de la création d'objets, séparation des responsabilités.  
- **Composite Pattern** : Traitement uniforme des objets individuels et des compositions.  
- **Observer Pattern** : Communication découplée entre objets (sujet-observateurs).  
- **Decorator Pattern** : Ajout de fonctionnalités dynamiquement sans modifier les classes.  
- **SOLID Principles** : Single Responsibility, Open/Closed, Liskov Substitution, Interface Segregation, Dependency Inversion.  
- **Design by Contract** : Préconditions, postconditions, invariants.  
- **Composition over Inheritance** : Préférer "has-a" à "is-a" pour plus de flexibilité.
